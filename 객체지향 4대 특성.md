## 객체지향의 4대 특성
객체 지향에는 4대 특성이 있다.<br>
#### 1. 추상화
추상화란 객체의 중요한 부분을 강조하기 위해 공통적인 속성과 기능을 추출하고 불필요한 세부사항을 제거하여 핵심만 표현하는 것이다.
* 복잡한 것은 숨김
* 필요한 정보만 외부에 보여줌<br><br>
사용하는 이유 : 코드를 이해하기 쉽게 하고, 설계를 단순하고 명확하게 하기 위해서
#### 2. 상속
상속이란 상위 클래스의 필드와 메서드를 하위 클래스가 물려받아 재사용하는 것이다.
* 부모 클래스 -> 자식 클래스
* 코드 재사용 가능<br><br>
사용하는 이유 : 중복 코드를 줄이고, 유지보수를 쉽게 하기 위해서
#### 3. 다향성
다향성이란 하나의 객체나 메서드가 상황에 따라 여러 형태로 동작할 수 있는 성질이다.
* 오버라이딩(Overriding): 부보 메서드를 자식이 재정의
* 오버로딩(Overloading): 같은 이름, 다른 매개변수<br><br>
사용하는 이유 : 하나의 코드로 다양한 동작을 처리하기 위해서
#### 4. 캡슐화(정보 은닉)
캡슐화란 데이터와 메서드를 하나의 객체로 묶고, 내부 구현을 숨긴 채 필요한 부분만 외부에 공개하는 것이다.<br><br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;사용하는 이유 : 데이터를 보호하고, 외부에서의 잘못된 접근을 방지하기 위해서
